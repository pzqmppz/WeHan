## 一、产品概述

### 1.1 产品定位

**武汉人才留汉智能服务平台**是一个连接武汉高校人才与本地企业的智能服务平台，旨在：
- 帮助武汉大学生**留在武汉**就业
- 帮助武汉企业**找到本地人才**
- 帮助政府**实现人才留汉政绩**

> **核心差异化**：不是招聘平台，而是"武汉大学生的就业AI顾问"

### 1.2 目标用户

| 用户群体 | 核心诉求 | 使用场景 |
|---------|---------|---------|
| **武汉大学生** | 找到合适的本地工作、获得求职指导 | C端 WeHan |
| **武汉企业** | 获得本地人才渠道、降低招聘成本 | B端 才聚江城（企业端）|
| **武汉高校** | 提高就业率、完成就业指标 | B端 才聚江城（学校端）|
| **武汉政府** | 人才留汉政绩可视化、政策触达 | B端 才聚江城（政府端）|

### 3.1 平台与技术要求

| 项目 | 要求 |
|-----|------|
| **MVP平台** | 扣子智能体 → 发布到豆包App |
| **选择理由** | 时间紧迫（3月1日截止），扣子+豆包月活已达重量级 |
| **后续迭代** | 根据用户反馈，再考虑H5/微信小程序开发 |

> **MVP策略说明**：
> - 扣子智能体可以直接发布到豆包App，无需额外开发
> - 微信小程序个人账号功能受限，企业账号需资质审核
> - 调试两套系统（小程序+B端）时间成本过高
> - 先用扣子快速验证产品价值，后续再投入原生开发

### 3.2 功能模块

#### 3.2.1 首页/岗位浏览

> **MVP独立策略**：C端用户可以自主创建岗位，无需等待B端数据同步
>
> ```
> 岗位来源（双通道）：
> ├── 通道1：平台岗位库（来自B端企业发布）
> │   └── 岗位列表、筛选、搜索、详情
> │
> └── 通道2：用户自定义岗位（MVP核心）⭐
>     ├── 分享导入：从招聘App分享链接/卡片到WeHan
>     ├── 粘贴导入：复制粘贴岗位JD文本
>     ├── AI解析：自动识别岗位名称、薪资、要求、企业信息
>     └── 创建成功：生成临时岗位，立即开始AI面试
> ```

| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| **自定义岗位** | 用户自主创建岗位，支持分享导入/粘贴导入 | P0 |
| **分享链接识别** | 识别招聘App分享链接，自动抓取岗位信息 | P0 |
| **JD粘贴识别** | 用户粘贴岗位JD文本，AI智能解析结构化 | P0 |
| **AI岗位解析** | 自动提取岗位名称、薪资、要求、企业信息 | P0 |
| 岗位列表 | 展示可用岗位（平台+用户自定义），支持分页加载 | P0 |
| 基础筛选 | 按行业/薪资/区域/应届友好度筛选 | P0 |
| 岗位搜索 | 关键词搜索岗位 | P1 |
| 岗位详情 | 展示岗位完整信息（JD、薪资、要求、企业信息） | P0 |
| 匹配度评分 | 展示用户与岗位的匹配度百分比 | P0 |
| 收藏岗位 | 收藏感兴趣的岗位 | P1 |

> **设计理念**：
> - C端体验独立完整，不依赖B端数据
> - 用户看到任何岗位（招聘网站/朋友圈/群分享），都可以立即开始面试练习
> - 后续与B端数据打通时，用户自定义岗位可关联到正式岗位

#### 3.2.2 求职Agent（核心模块）

> **Agent核心能力**：WeHan的AI面试Agent不是简单的"对话+打分"，而是具备真正的智能体能力：
>
> | 能力 | 描述 |
> |-----|------|
> | **目标理解** | 基于用户简历 + 岗位JD，智能理解面试考察目标 |
> | **问题生成** | 自主生成针对性的面试问题框架（10-20题） |
> | **自主决策** | 根据用户回答实时决定：追问/跳过/拓展/调整难度 |
> | **动态调整** | 面试过程中根据表现调整节奏和问题方向 |
> | **综合评估** | 多维度分析生成个性化评估报告 |
>
> 这种"根据情境自主决策"的能力，是区别于普通聊天机器人的核心特征。

| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| **简历上传** | 支持PDF/Word/图片格式上传 | P0 |
| **AI简历解析** | 自动提取技能、经历、教育、项目信息 | P0 |
| **人才画像生成** | 基于简历生成结构化用户画像 | P0 |
| **自定义岗位导入** | 支持分享链接/粘贴JD创建岗位 | P0 |
| **岗位JD智能解析** | AI自动识别岗位名称、薪资、要求、企业信息 | P0 |
| **岗位匹配推荐** | 基于画像推荐匹配岗位（平台库+自定义）| P0 |
| **AI面试模拟** | 对话流式面试，AI自主决策追问方向 | P0 |
| **面试大纲预览** | 面试前展示AI生成的题目框架 | P1 |
| **语音输入** | 支持语音回答（ASR转文字） | P1 |
| **评估报告** | 综合评分 + 多维度评估 + 改进建议 | P0 |
| **一键投递** | 随时可投递，携带面试报告和匹配度 | P0 |
| **投递状态追踪** | 查看投递状态（待处理/已查看/面试邀约/已录用） | P0 |

> **自定义岗位面试流程**：
> ```
> 用户操作：
> ├── 方式1：分享导入
> │   ├── 在招聘App看到心仪岗位
> │   ├── 点击分享 → 选择WeHan
> │   └── AI自动解析岗位信息，生成面试大纲
> │
> ├── 方式2：粘贴导入
> │   ├── 复制岗位JD文本
> │   ├── 在WeHan粘贴
> │   └── AI自动解析，提取关键信息
> │
> └── 方式3：手动输入
>     ├── 输入岗位名称、公司名称
>     ├── 可选：粘贴JD详情
>     └── AI补充完善岗位信息
>
> AI处理：
> ├── 解析JD → 提取岗位名称、薪资、要求、技能
> ├── 生成岗位标签 → 行业、岗位类型、难度等级
> ├── 创建面试大纲 → 基于JD生成10-20道面试题
> └── 保存为"我的岗位" → 可重复练习
> ```

### 3.3 核心流程

#### AI面试模拟流程

```
Step 1: 简历解析
├── 用户上传简历
└── AI解析生成人才画像

Step 2: 岗位匹配
├── AI推荐匹配岗位
└── 用户选择目标岗位

Step 3: 面试大纲生成
├── AI根据岗位JD生成面试题（10+题）
├── 融入企业上传的自定义问题
└── 用户预览题目列表

Step 4: 对话流面试
├── AI逐题提问（支持语音/文字）
├── AI根据回答追问/拓展
├── 实际问题数可能从10道拓展到20道
└── 用户可暂停/继续

Step 5: 评估报告
├── 综合评分（0-100分）
├── 多维度评估：表达能力、专业能力、逻辑思维、应变能力
├── 优秀回答回顾
├── 待改进点 + 改进建议
└── 推荐学习资源

Step 6: 一键投递
├── 随时可投递，评分作为建议展示（不拦截）
├── 展示匹配度："你的匹配度：75%，建议提升面试能力"
├── 一键投递，携带面试对话摘要 + 亮点片段
└── 投递后追踪状态
```

#### 评分与投递机制

> **核心原则**：评分是工具不是权力——AI没有资格拦截学生的机会，只能给建议

```
评分展示：
├── 匹配度评分：展示用户与岗位的匹配度百分比
├── 面试能力评分：展示面试表现评估（0-100分）
└── 改进建议："建议提升XX能力，可提高匹配度"

投递逻辑：
├── 学生可随时投递，无门槛限制
├── 投递时携带：简历 + 面试报告 + 匹配度
└── 企业端查看：收到完整材料，自主判断

设计理念：
├── 评分是建议，不是拦截
├── 让学生看到自己的位置，但不剥夺机会
└── 企业收到更多信息，做出更好判断
```

## 五、数据互通规范

### 5.1 核心数据实体

| 实体 | 字段 | 数据流向 |
|-----|------|---------|
| **岗位** | ID、标题、薪资、地点、JD、要求、企业ID、状态 | B端创建 → C端展示 |
| **用户** | ID、姓名、学校、专业、简历、画像 | C端创建 → B端查看 |
| **投递** | ID、用户ID、岗位ID、面试分数、报告、状态 | C端创建 → B端接收 |
| **政策** | ID、标题、内容、类型、生效时间 | B端创建 → C端展示 |
| **企业** | ID、名称、行业、规模、认证状态 | B端创建 → C端展示 |

### 5.2 API接口规划

| 模块 | 接口 | 方向 |
|-----|------|------|
| **岗位** | 获取岗位列表、获取岗位详情 | B → C |
| **用户** | 用户注册、用户登录、更新简历 | C端 |
| **面试** | 创建面试会话、提交回答、获取评估报告 | C端 |
| **投递** | 创建投递、获取投递列表、更新投递状态 | 双向 |
| **政策** | 获取政策列表 | B → C |
| **统计** | 获取留汉指数、获取就业数据 | B端 |

### 5.3 数据存储策略

| 数据类型 | 存储策略 | 说明 |
|---------|---------|------|
| 已投递面试 | 永久保存 | 完整对话 + 语音 + 报告 |
| 未投递面试 | 24h后删除语音 | 对话摘要保留7天 |
| 用户画像 | 永久保存 | 用于匹配推荐 |
| 岗位数据 | 永久保存 | 下架后归档 |

## 六、MVP功能范围

### 6.1 MVP策略

- **C端做深**：求职Agent + 心理健康Agent（差异化亮点）
- **B端四端联动**：体现平台完整愿景
- **数据准备**：30条岗位 + 5-10家企业 + 5-10条政策

### 6.2 MVP功能清单

#### C端 MVP

| 模块 | 功能 | 状态 |
|-----|------|------|
| 首页 | 岗位列表、筛选、详情、匹配度 | ✅ 必做 |
| 求职Agent | 简历解析、AI面试、评估报告、投递 | ✅ 必做 |
| 心理健康Agent | 求职焦虑咨询、情绪记录 | ✅ 必做 |
| 个人中心 | 个人信息、简历、投递记录 | ✅ 必做 |
| 社区 | 学长带路、互动社区 | ❌ 暂不做 |
| 政策服务 | 政策一键兑现、驿站联动 | ❌ 暂不做 |
| 职业测评 | 性格测评、试岗体验 | ❌ 暂不做 |
| 积分系统 | 积分、等级 | ❌ 暂不做 |

#### B端 MVP

| 模块 | 功能 | 状态 |
|-----|------|------|
| 门户页面 | 产品介绍、政策滚动、数据展示 | ✅ 必做 |
| 企业端 | 岗位发布、人才库、查看报告 | ✅ 必做 |
| 政府端 | 留汉指数大屏、政策发布 | ✅ 必做（简化） |
| 学校端 | 就业看板、岗位推送 | ✅ 必做（简化） |
| 管理员 | 内容管理、审核、用户管理 | ✅ 必做 |

### 6.3 开发优先级

```
P0（必须完成）：
├── C端 AI面试模拟（核心卖点）
├── C端 简历解析 + 评估报告
├── C端 岗位浏览 + 投递
├── B端 门户页面
├── B端 管理员后台
└── B端 企业端基础功能

P1（重要功能）：
├── C端 心理健康Agent
├── B端 政府端数据大屏
└── B端 学校端看板

P2（加分功能）：
├── 简历匿名化
├── 政策滚动播放
└── 语音输入（ASR）
```
