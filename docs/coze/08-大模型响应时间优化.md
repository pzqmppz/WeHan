# 大模型响应时间优化指南

> 本文档介绍如何通过减少输出 Token 数量，直接降低大模型响应时间。

---

## 一、核心理论：响应时间公式

大模型响应时间的计算公式：

```
latency（响应时间） = TTFT + output_token_num × TPOT
```

| 参数 | 说明 | 可优化性 |
|-----|------|---------|
| **TTFT** | 首个 Token 响应时间 | ❌ 由模型架构/部署环境决定 |
| **output_token_num** | 输出 Token 数量 | ✅ **唯一可通过业务层优化的指标** |
| **TPOT** | 首 Token 后每个 Token 耗时 | ❌ 由模型/部署环境决定 |

**结论**：减少输出 Token 数量是降低响应时间最直接、最高效的方式。

---

## 二、四大优化手段

### 2.1 压缩方法名和参数

**做法**：精简工作流/插件的方法名、参数名，用短字符替代长命名。

**示例**：
```
优化前：getWeather → 3 tokens
优化后：gw        → 1 token

优化前：getCountry → 3 tokens
优化后：gc         → 1 token
```

**适用场景**：工具调用、函数命名

---

### 2.2 自定义编码

**做法**：将意图识别、功能选择的结果用数字/符号替代文字描述。

**示例**：
```
优化前：联网搜索 → 2 tokens
优化后：1       → 1 token

优化前：英语陪聊 → 2 tokens
优化后：2       → 1 token
```

**适用场景**：意图识别、多功能选择

---

### 2.3 摒弃 JSON/XML 嵌套结构（效果最显著）

**做法**：用管道符（|）、CSV 等轻量格式替代 JSON/XML 的嵌套键值对。

**示例**：
```
优化前（JSON，48 tokens）：
{"func":"getWeather","args":{"city":"shenzhen","date":"2024-12-12"}}

优化后（管道符，17 tokens）：
getWeather|shenzhen|2024-12-12
```

**优化效果**：Token 减少 65%+

**适用场景**：参数传递、结果返回

---

### 2.4 中文数字替代阿拉伯数字

**做法**：用中文数字表达日期/数值，替代阿拉伯数字+符号的组合。

**示例**：
```
优化前：2025-03-20 → 10 tokens
优化后：二五年三月二十日 → 5 tokens
```

**适用场景**：日期、简单数值输出

---

## 三、优化手段对比

| 优化手段 | Token 优化效果 | 实现难度 | 适用场景 |
|---------|---------------|---------|---------|
| 压缩方法名 | 中等 | 低 | 工具调用、参数命名 |
| 自定义编码 | 中等 | 低 | 意图识别、功能选择 |
| 摒弃 JSON 嵌套 | **最显著** | 中 | 参数传递、结果返回 |
| 中文数字替代 | 一般 | 低 | 日期、数值输出 |

## 四、实战案例：天气参数提取

**场景**：从用户输入中提取城市和日期信息

### 优化前

**系统提示词**：
```
提取天气参数，输出 JSON 格式，时间用阿拉伯数字
```

**大模型输出**：
```json
{"city":"深圳","start_time":"2025-03-20","end_time":"2025-03-20"}
```

**消耗**：47 tokens，响应时间 1175ms（TPOT=25ms）

### 优化后

**系统提示词**：
```
提取天气参数，输出格式：城市|开始时间|结束时间
时间用中文数字，示例：深圳|二五年三月二十日|二五年三月二十日
```

**大模型输出**：
```
深圳|二五年三月二十日|二五年三月二十日
```

**消耗**：13 tokens，响应时间 325ms

### 对比结果

| 指标 | 优化前 | 优化后 | 降幅 |
|-----|-------|-------|------|
| Token 数 | 47 | 13 | **72%** |
| 响应时间 | 1175ms | 325ms | **72%** |

## 五、落地建议

### 5.1 优化原则

在保证业务能解析结果的前提下，**去掉所有冗余的字符、格式、命名**：
- JSON 的大括号、键名
- XML 的标签
- 长方法名
- 阿拉伯数字的连接符

### 5.2 场景适配

| 场景 | 推荐优化手段 |
|-----|-------------|
| 工具调用/参数传递 | 压缩方法名 + 管道符格式 |
| 意图识别/功能选择 | 自定义编码 |
| 日期/数值输出 | 中文数字替代 |

### 5.3 Coze 工作流配置

这些优化手段可直接应用于 Coze 工作流的**大模型节点配置**：

1. 打开大模型节点的提示词设置
2. 修改输出格式要求（如：要求输出管道符格式）
3. 添加输出示例引导模型

### 5.4 工具辅助

使用**方舟 Token 计算器**（火山引擎在线工具）预估 Token 数量，做到有数据支撑的优化。

## 六、总结

| 要点 | 说明 |
|-----|------|
| 核心逻辑 | 牺牲一点输出"可读性"，换取大幅响应时间提升 |
| 最佳手段 | **摒弃 JSON 嵌套结构，使用管道符格式** |
| 实现成本 | 仅修改提示词，无需改动模型/部署 |
| 收益 | 响应时间可降低 50%-70% |
