# 扣子插件开发

## 概述

插件扩展了智能体的能力边界，让智能体可以调用外部 API、处理文件、生成图片等。

## 插件类型

### 1. 官方插件
扣子提供的即用型插件：
- 头条新闻
- 天气查询
- 图片搜索
- 网页读取
- 等 100+ 官方插件

### 2. 自定义插件
开发者自己创建的插件，支持：
- 基于 API 创建
- 使用 IDE 创建
- JSON/YAML 导入
- 代码注册

## 创建自定义插件

### 方式一：基于 API 创建

1. 进入插件中心
2. 点击「创建插件」
3. 选择「基于 API 创建」
4. 配置 API 信息：
   - API 地址
   - 请求方法
   - 请求参数
   - 响应格式

### 方式二：导入 cURL

HTTP 请求节点支持导入 cURL 命令：

```bash
curl --location --request POST 'https://api.example.com/endpoint' \
--header 'Authorization: Bearer YOUR_TOKEN' \
--header 'Content-Type: application/json' \
--data '{ "key": "value" }'
```

### 方式三：导入 OpenAPI YAML

扣子支持导入 OpenAPI 3.0 规范的 YAML 文件，自动生成插件定义。

**WeHan 项目提供的 OpenAPI 规范**:
- `client/openapi/question_plugin.yaml` - 多轮问询控制器插件

## 调用付费插件

## 插件配置项

| 配置项 | 说明 |
|-------|------|
| 插件 ID | 唯一标识 |
| 插件名称 | 显示名称 |
| 描述 | 功能说明 |
| API 地址 | 请求 URL |
| 请求方法 | GET/POST/PUT/DELETE |
| 请求参数 | 输入参数定义 |
| 鉴权方式 | API Key / OAuth / 自定义 |
| 超时时间 | 默认 120s |

## HTTP 请求节点详解

### 鉴权配置

| 类型 | 说明 |
|-----|------|
| Bearer Token | OAuth 2.0 令牌 |
| 自定义 | Key-Value 形式 |
| 无需鉴权 | 公开 API |

### 请求体格式

| 格式 | 说明 |
|-----|------|
| form-data | 表单数据 |
| x-www-form-urlencoded | URL 编码 |
| JSON | JSON 格式 |
| raw text | 纯文本 |

### 异常处理

- **超时设置**：默认 120s，最长 600s
- **重试次数**：默认 3 次，最大 10 次
- **异常忽略**：开启后节点失败不中断工作流

## WeHan 多轮问询插件

### 背景

由于扣子智能体的变量记忆机制不够可靠（在长对话中可能丢失状态），我们设计了基于数据库的多轮问询状态管理方案。

### 设计思路

```
┌─────────────────────────────────────────────────────────┐
│                   多轮问询状态管理                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  工作流一 (generate_questions_workflow)                  │
│       │                                                 │
│       │ 生成问题数组                                     │
│       ▼                                                 │
│  ┌─────────────────────────────────────┐               │
│  │  POST /api/coze/question/init       │               │
│  │  Body: { session_id, questions[] }  │               │
│  └─────────────────┬───────────────────┘               │
│                    │                                    │
│                    ▼ 返回第一题                         │
│  ┌─────────────────────────────────────┐               │
│  │  智能体对话循环                      │               │
│  │  - 提问                              │               │
│  │  - 接收回答                          │               │
│  │  - POST /api/coze/question/next     │               │
│  │  - 获取下一题或完成信号              │               │
│  └─────────────────┬───────────────────┘               │
│                    │                                    │
│                    ▼ 所有问答完成                       │
│  ┌─────────────────────────────────────┐               │
│  │  返回完整问答数据                    │               │
│  │  { questions[], answers[] }         │               │
│  └─────────────────────────────────────┘               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 数据库表设计

**CozeQuestionSession 表**:

| 字段 | 类型 | 说明 |
|-----|------|------|
| sessionId | String | Coze 会话 ID（主键） |
| questions | Json | 问题数组 `string[]` |
| answers | Json | 回答数组 `string[]` |
| currentIdx | Int | 当前问题索引 |
| isFinish | Boolean | 是否已完成 |
| expiredAt | DateTime | 过期时间（30分钟） |

### API 接口

#### 1. 初始化会话

```
POST /api/coze/question/init
X-API-Key: your-api-key

{
  "session_id": "coze_conversation_xxx",
  "questions": [
    "请介绍一下你自己",
    "你为什么选择这个岗位",
    "你的优势是什么"
  ]
}
```

**响应**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "current_question": "请介绍一下你自己",
    "current_idx": 0,
    "is_finish": false,
    "total_questions": 3
  }
}
```

#### 2. 记录回答并获取下一题

```
POST /api/coze/question/next
X-API-Key: your-api-key

{
  "session_id": "coze_conversation_xxx",
  "answer": "我叫张三，是一名应届毕业生..."
}
```

**响应（继续）**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "current_question": "你为什么选择这个岗位",
    "current_idx": 1,
    "is_finish": false
  }
}
```

**响应（完成）**:
```json
{
  "code": 200,
  "msg": "all questions finished",
  "data": {
    "is_finish": true,
    "questions": ["Q1", "Q2", "Q3"],
    "answers": ["A1", "A2", "A3"]
  }
}
```

#### 3. 强制结束会话

```
POST /api/coze/question/force-finish
X-API-Key: your-api-key

{
  "session_id": "coze_conversation_xxx"
}
```

**响应**:
```json
{
  "code": 200,
  "msg": "会话已强制结束",
  "data": {
    "is_finish": true,
    "questions": ["Q1"],
    "answers": ["A1"],
    "unfinished_questions": ["Q2", "Q3"],
    "total_questions": 3,
    "completed_count": 1
  }
}
```

### 在扣子中配置插件

#### 步骤 1: 导入 OpenAPI 规范

1. 进入扣子插件中心
2. 点击「创建插件」
3. 选择「导入 OpenAPI」
4. 上传 `client/openapi/question_plugin.yaml` 或粘贴内容
5. 确认导入

#### 步骤 2: 配置鉴权

1. 在插件设置中选择「API Key」鉴权
2. Header 名称: `X-API-Key`
3. 值: 从 B 端获取的 `OPEN_API_KEY`

#### 步骤 3: 在工作流中使用

**工作流一 (generate_questions_workflow)**:
1. 大模型节点生成问题数组
2. 调用 `initSession` 接口初始化会话
3. 输出第一题给智能体

**智能体对话循环**:
1. 接收用户回答
2. 调用 `nextQuestion` 接口
3. 判断 `is_finish`:
   - `false`: 继续提问
   - `true`: 进入评估阶段

**异常处理**:
- 用户说「结束面试」「不想回答了」时
- 调用 `forceFinish` 接口
- 返回已完成的数据

### OpenAPI 规范文件位置

```
WeHan/
└── client/
    └── openapi/
        └── question_plugin.yaml   # 多轮问询插件规范
```

### 错误码说明

| 状态码 | 说明 |
|-------|------|
| 200 | 成功 |
| 400 | 参数错误（缺少必填项等） |
| 401 | API Key 无效 |
| 404 | 会话不存在 |
| 410 | 会话已过期（超过30分钟） |
| 500 | 服务器内部错误 |

## 插件权限

- **个人空间插件**：仅能被个人调用
- **团队空间插件**：能被任意团队成员调用
- **版本管理**：发布新版本后，使用该插件的智能体自动使用最新版本

## 最佳实践

1. **API 设计**：保持接口简洁，参数明确
2. **错误处理**：返回清晰的错误信息
3. **文档完善**：提供详细的使用说明
4. **版本控制**：合理管理插件版本
5. **状态管理**：对于多轮对话场景，使用数据库持久化状态
